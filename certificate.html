Welcome to the Development Certification Process

For smooth integration and stakeholder satisfaction, Triple Play Pay has built this quick certification process that utilizes a test API Key and specified tests in our Sandbox environment to ensure that developers have the resources and know-how to properly utilize our endpoints, requests, and responses for successful transactions and user experiences.

Work through these steps for your successful integration of Triple Play Pay’s API.

You will accomplish:

  1. Familiarizing yourself with the Developer Documentation and GitHub repository
  2. Integrating the API by using a Sandbox test API key
  3. Submitting our required test data
  4. Receiving a certificate of completion

Setup

To begin the Developer Certification process for our Payment Processing API, follow the steps outlined below:

  1. The business must enroll as an Independent Software Vendor (ISV) or Merchant by visiting tripleplaypay.com/partner and then enrolling the lead developer once approved.
  2. The developer should then complete Development Certification registration at tripleplaypay.com/api/registration.
  3. Upon completion, review the requirements provided in the Certification Instructions email. You can also explore the API documentation and access the source code on our GitHub repository.
  4. Log in to the Sandbox testing environment at sandbox.tripleplaypay.com using the provided credentials. Retrieve your unique API key from the account's details.
  5. Begin testing the API by integrating it into your platform using the URL sandbox.tripleplaypay.com/api. You can refer to the sample code available on our GitHub repository.
  6. Conduct the tests specified in the Tests section of this page to ensure the functionality and compatibility of the API integration.
  7. Once all tests have been successfully passed, you will receive a certificate of completion for the Developer Certification process.
